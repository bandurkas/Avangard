# Архитектурный Аудит и План Развития: Avangard Fleet Management

Вы абсолютно правы: чтобы система стала по-настоящему Enterprise-уровня (согласно нашему CTO-аудиту), недостаточно просто выводить список машин. Нам необходима глубокая детализация по каждому юниту и мощный финансовый блок (Payroll).

Ниже представлен обновленный план реализации, который покрывает ваши новые требования и сверяется с изначальным аудитом.

## 1. Сверка с CTO-Аудитом: Что сделано, а что предстоит

В нашем изначальном CTO-аудите мы выделили 4 фазы. Вот текущий статус:

*   ✅ **Фаза 1: Фундамент (Выполнено).** Мы успешно ушли от MVP на `localStorage` и перевели систему на связку React Query + Standalone API (Express/PostgreSQL). UI стал реактивным и готовым к production-нагрузкам. Внедрена базовая Телеметрия в симулятор.
*   ⏳ **Фаза 2: Telemetry Ingestion Pipeline (В планах).** RabbitMQ/Kafka для приема GPS-треков. Пока мы симулируем этот поток на фронтенде, но для production потребуется брокер.
*   ⏳ **Фаза 3: Mobile Offline-First (В планах).** Перевод симулятора в реальное React Native приложение (SQLite/WatermelonDB).
*   🚀 **Фаза 4: Smart Payroll & Analytics (В ФОКУСЕ СЕЙЧАС).** Вы как раз запросили реализацию этого блока: детальная аналитика авто и расчет зарплат.

## 2. Ответы на ваши вопросы (Open Questions)

### Средний расход топлива (В панели администратора)
**Как это должно работать:** 
В дашборде администратора (Overview) "Средний расход" должен показывать **агрегированную метрику по всему активному автопарку** (например, "32 л / 100 км по парку"), чтобы оценивать общую эффективность.
Однако, в *детальном профиле автомобиля* (о котором вы просите ниже), мы будем показывать **персональный расход конкретной машины**, чтобы выявлять аномалии (например, "МАЗ №3 жрет 45 л / 100 км — возможен слив топлива").

## 3. Proposed Changes: Новые Интерфейсы (UI/UX)

### [NEW] Карточка Детализации Автомобиля (Vehicle Details Modal/Drawer)
Когда диспетчер кликает на машину в списке или на карте, справа должна выезжать красивая панель (Slide-over / Drawer), содержащая всю подноготную по машине "в смене":
1.  **Header:** Модель ТС, Госномер, Статус (В рейсе / На базе / Ремонт), текущий Водитель с фото.
2.  **Live-Метрики:** 
    *   Остаток топлива (прогресс-бар).
    *   Текущая скорость и статус зажигания.
    *   Температура рефрижератора (если применимо).
3.  **Аналитика Текущей Смены:**
    *   Время старта смены.
    *   Пройденная дистанция (км).
    *   Соотношение "В движении" vs "Простой" (круговая диаграмма).
4.  **События (Log):** Лента остановок, заправок и съездов с маршрута.

### [NEW] Раздел "Расчет Оплаты" (Smart Payroll Module)
Мы добавим новую вкладку в левое меню: **"Расчеты (Payroll)"**.
1.  **Таблица Зарплат (Billing Table):** 
    *   Список всех водителей за выбранный период (например, Июнь 2026).
    *   Столбцы: `Водитель`, `Часы в пути`, `Часы простоя`, `Ставка (руб/час)`, `Бонусы`, `Штрафы (пережог топлива)`, `ИТОГО К ВЫПЛАТЕ`.
2.  **Payroll Analytics:**
    *   Графики: ФОТ (Фонд оплаты труда) по дням/неделям.
    *   Отчеты по переработкам (Overtime).
